## @HostListener — no event argument in metadata when method takes no params

When using `@HostListener('window:resize', ['$event'])` but the method signature is `onResize(): void` (no parameter), TypeScript strict mode raises: `TS2554: Expected 0 arguments, but got 1`.

**Fix:** Either remove `['$event']` from the decorator metadata, or add the `event: Event` parameter to the method.

**Why:** Angular 21 with strict TypeScript enforces that the argument list in the decorator metadata matches the actual method arity.

**How to apply:** Always match @HostListener metadata args to the method parameter count. For resize handlers that only read `window.innerWidth`, drop the event parameter entirely.

## zone.js not included by default in Angular 21 CLI scaffolding

Angular 21 projects generated by the CLI do not include `zone.js` in `package.json` by default (Angular is moving toward zoneless). Adding `"polyfills": ["zone.js"]` to `angular.json` without installing the package causes a build error.

**Fix:** Run `npm install zone.js --save` before adding it to polyfills.

**How to apply:** Check `node_modules/zone.js` exists before referencing it in angular.json.

## Bundle size warning — PrimeNG stylesheet

PrimeNG's compiled CSS is ~365 kB, which exceeds the default Angular 500 kB initial bundle budget. This produces a warning but NOT an error. It is expected for a full PrimeNG integration.

**How to apply:** Adjust the budget in angular.json `production.budgets` if the warning is undesirable, but do not treat it as a blocking error.

## PrimeNG v21 — renamed and restructured components

PrimeNG v21 renamed and reorganized several commonly used components. Old module-based imports no longer work.

**Key renames:**
- `DropdownModule` / `p-dropdown` → `SelectModule` / `p-select` from `primeng/select`
- `CalendarModule` / `p-calendar` → `DatePickerModule` / `p-datepicker` from `primeng/datepicker`
- `MessageModule` → standalone `Message` class from `primeng/message`

**Event API change for `p-select`:**
- Does NOT emit via `(ngModelChange)` — use `(onChange)="handler($event.value)"` instead
- Still supports `[ngModel]` for one-way binding (needs `FormsModule` imported)
- For reactive forms, use `formControlName` directly

**NG8107 warning:** When the Project interface defines `center` as non-nullable `{ id; name; acronym }`, using `p.center?.acronym` in templates triggers NG8107. Use `p.center.acronym` (non-optional) instead.

**How to apply:** Always use `primeng/select` and `primeng/datepicker` for dropdown and date fields. Check available exports via `node_modules/primeng/package.json` exports field.
